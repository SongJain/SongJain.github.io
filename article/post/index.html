响应流程

客户端发送 Http 请求后，先到 tmocat 引擎中进行解析资源，在把这些请求信息封装到 request 中，同时，response 作为一个空参数和 reques 一起传入doGet() 方法中，方法中有对 response 中的内容进行填写。方法结束后，tomcat 引擎从response 缓存中拿取信息，并且跟自己的内容组装成 Http 响应（响应行，响应头，响应体）。

文件下载

文件下载的实质就是文件拷贝，将文件从服务器端拷贝到浏览器端。所以文件下载需   要IO技术将服务器端的文件使用 InputStream 读取到，在使用 ServletOutputStream 写到 response 缓冲区中。

Q：什么时候会文件下载？？？

A：浏览器不能解析的文件就下载！！

注意的点：

1、response 的流不需要手动关闭，Tomcat 会帮我们关闭。

2、getOutputStream() 和 getWriter() 不能同时被调用。
